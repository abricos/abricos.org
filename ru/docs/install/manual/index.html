Ручная установка

Установка платформы Абрикос на свой сайт из репозитария https://github.com/abricos

1. Произведите экспорт ядра платформы из репозитария Github (https://github.com/abricos/abricos-core/) в любое удобное для вас место, например c:\abricos


	cd [путь к корневой директории сайта]
	git clone https://github.com/abricos/abricos-core.git .

Примечание: скачать ядро можно так же по прямой ссылке https://github.com/abricos/abricos-core/archive/master.zip. В данном случае в архиве будет создана папка abricos-core-master из которой файлы должны быть перемещены в корень сайта.

2. Произведите экспорт всех необходимых модулей из Github https://github.com/abricos в папку modules ядра платформы Абрикос 


	cd [путь к корневой директории сайта]/modules
	git clone https://github.com/abricos/abricos-mod-[имя модуля].git

Примечание: скачать каждый модуль можно так же по прямой ссылке, например модуль "Блоги" - https://github.com/abricos/abricos-mod-blog/archive/master.zip. В данном случае в архиве будет создана папка abricos-mod-blog, которую нужно извлеч в папку modules платформы Абрикос и переименовать папку abricos-mod-blog в папку blog.

Примечание. Обратите внимание на зависимости в модулях. Зачастую для работы модуля необходимо наличие других модулей. Информация о модулях и их зависимостей можно просмотреть здесь.

3. В папке платформы переименуйте файл def.htaccess в .htaccess

4. Зайдите в папку платформы includes, переименуйте файл config.example.php в config.php и отредактируйте его указав в нем доступ к базе данных

5. Выгрузите файлы платформы Абрикос на сервер в корневой каталог вашего сайта

6. Установите права на запись папке cache

7. Откройте заглавную страницу вашего сайта.
Примечание. При первом открытии страницы сайта на платформе Абрикос может быть значительная задержка по времени. Это связанно с тем, что в этот момент платформа создает необходимые для своей работы таблицы в базе данных.

Если все сделано правильно, будет создана и отображена заглавная страница вашего сайта.

Если заглавная страница не отображена, просмотрите список часто задаваемых вопросов. Возможно проблема и ее устранение уже описана в этом разделе.
